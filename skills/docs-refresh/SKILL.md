# Docs Refresh

用这个技能写或更新文档。文档要帮助真实读者完成任务，而不是记录过程流水账。

## Trigger

当用户说以下任一类需求时使用：

- “写个 README/文档/说明”“整理成文档”“更新 docs”
- “写运行手册/迁移指南/FAQ/架构说明”
- “给我一份可以贴到公共文档里的内容”

## Workflow

1. **Identify Audience**
   - 读者是谁：用户、开发者、运维、评审者、未来维护者。
   - 读者要完成什么任务。

2. **Collect Facts**
   - 从代码、配置、测试、命令输出或官方来源收集事实。
   - 不把未验证猜测写成确定说明。

3. **Choose Shape**
   - README：目标、快速开始、核心概念、常用命令、故障排查。
   - Runbook：前置条件、步骤、验证、回滚、告警和联系人。
   - Architecture：结论、边界、组件、数据流、取舍、风险。
   - User Guide：任务路径、输入、反馈、常见错误。

4. **Write For Action**
   - 命令给完整 `cd` 路径和必要环境变量。
   - 概念说明少而准。
   - 避免历史过程、临时草稿和无关背景。

5. **Validate**
   - 检查路径、命令、参数、文件名和链接是否真实存在。
   - 能运行的命令尽量运行；不能运行则说明未验证。

6. **Keep Current**
   - 删除或更新过时入口。
   - 如果有索引或目录图，同步更新。

## Output

默认输出成可直接落文件的 Markdown。

如只需总结，使用：

```markdown
## What Changed

## How To Use

## Verification

## Notes
```

## Quality Bar

- 文档必须面向读者任务，不是作者过程日志。
- 命令、路径、参数必须可复制。
- 对不确定或未验证内容要标注。
