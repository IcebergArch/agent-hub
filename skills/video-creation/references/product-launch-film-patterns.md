## Best Outline: 实事求是

视频创作的最高纲要是“实事求是”。它不是保守，而是让创意建立在真实材料、真实主题和真实能力之上：

- **真实素材**：先看用户给的截图、视频、页面、产品资料和可访问 artifact，再写镜头。
- **真实主题**：产品、人物、品牌、事件、功能、教程或概念各有不同核心，不要把它们拍成同一种片。
- **真实风格**：叙事、镜头语言、光线、色彩、节奏、声音和代入感必须服务这条片的类型，不套模板。
- **真实场景**：先判断这是产品宣传、教程、case demo、发布片、纪录片式讲述、品牌短片、概念片、平台能力验证还是 blocked 导演稿，再选结构。
- **真实能力**：平台能生成什么、不能生成什么、单 clip 时长、timeline 是否可串联、音频和字幕是否可控，都要先确认。
- **真实结果**：最终产物以媒体探针和视觉检查为准，不以工具口头声明为准。

实事求是的反面不是“创意”，而是“漂浮”：抽象光效、随机 UI、虚构能力、没有验收的成功声明、为了通用而抹掉具体场景。

## Creative Variables

任何视频在写 shot list 前，先明确这些变量。它们决定这条片长什么样，而不是由“高级感”三个字决定。

| Variable | Questions |
| --- | --- |
| 视频类型 | 产品宣传、教程、发布片、品牌短片、纪录片、case demo、概念片、社媒短视频、能力验证？ |
| 观众关系 | 观众是第一次认识、准备学习、准备购买、内部评审，还是要被情绪打动？ |
| 叙事结构 | 线性教程、问题-解决、从宏观到微观、人物带入、能力 montage、悬念揭示、before/after？ |
| 镜头语言 | 慢推、横移、手持、宏观微距、俯视、主观视角、屏幕录制、空间层叠、快切 montage？ |
| 光线 | 暗场发布、自然光纪录、明亮办公、舞台 spotlight、冷暖对比、低调高反差、柔光可信？ |
| 色彩 | 品牌色克制点缀、真实环境色、黑白纪实、低饱和高级、明亮协作感、强对比社媒冲击？ |
| 节奏 | 克制、递进、快节奏、教程式停顿、情绪起伏、发布片 build-up、社媒前 3 秒抓人？ |
| 声音 | 旁白、无旁白音乐驱动、UI sound design、采访同期、环境声、节拍点转场？ |
| 代入感 | 通过人、任务、问题、视角、UI 操作、空间运动，还是产物结果让观众进入？ |
| 信息密度 | 大字少信息、教程步骤、产品 UI 细节、业务上下文、情绪画面、结尾 CTA？ |

这些变量可以组合出不同风格。不要为了泛化把它们压扁成同一个“高级科技风”。

## Scenario Outlines

不要把所有视频都套成同一种结构。先选当前任务的纲要：

### 产品宣传片

目标：观众先记住产品是什么、在哪工作、为什么可信。

结构：

1. 产品名和总览。
2. 核心入口。
3. 能力链路。
4. 可信控制点。
5. 产物和结果。
6. 系统总览和品牌收束。

重点：case 只是承载，不是唯一目的；每个镜头都要提升产品认知。

### 品牌短片

目标：观众记住品牌气质、价值观或情绪主张。

结构：

1. 情绪或命题开场。
2. 人、场景或物的真实细节。
3. 品牌价值与具体行动连接。
4. 情绪峰值或一句主张。
5. 品牌标识收束。

重点：不需要密集展示功能，但不能空喊价值观；要靠真实细节建立信任。

### 纪录片式案例

目标：让观众通过人物、问题和真实过程相信结果。

结构：

1. 人或团队登场。
2. 问题和限制。
3. 过程、选择和挫折。
4. 工具或方法介入。
5. 结果和反思。

重点：镜头可以更生活化、手持或自然光；代入感来自真实过程，而不是炫技。

### 概念片 / 愿景片

目标：表达未来方向或抽象理念，但仍需要可理解的锚点。

结构：

1. 核心问题或愿景。
2. 抽象意象与真实对象交替。
3. 能力或理念逐步显形。
4. 回到可理解的产品、场景或行动。

重点：允许更诗意，但必须有真实锚点；不能全片漂浮。

### 社媒短视频

目标：在极短时间内抓注意力并传达一个记忆点。

结构：

1. 前 1-3 秒强 hook。
2. 一个清晰问题或反差。
3. 快速展示解决动作或结果。
4. 简短 CTA。

重点：节奏可以快，但不要牺牲核心可读信息；一个短视频只讲一个主点。

### 轻教程视频

目标：观众看完知道怎么做一件事。

结构：

1. 任务目标。
2. Step 1 入口。
3. Step 2 添加上下文。
4. Step 3 选择或触发能力。
5. Step 4 review/refine。
6. Step 5 export/approve。

重点：步骤字幕和 UI 操作路径优先，镜头运动要克制，不能妨碍学习。

### Case Demo

目标：用一个具体业务 case 证明产品能力。

结构：

1. Case brief。
2. 输入材料。
3. Agent 执行过程。
4. 中间状态和工具调用。
5. 多个 artifact 结果。
6. 人工审查和下一步。

重点：保留业务具体性，不要泛化成空洞的“AI 生成一切”；但不要把 case 写进通用规则。

### 发布片 / Launch Film

目标：建立发布质感和产品愿景，同时不脱离真实产品。

结构：

1. 高识别度开场。
2. 产品空间或工作台建立。
3. 三到五个核心能力段落。
4. 节奏上扬的 proof montage。
5. 静下来给出产品名和一句主张。

重点：可以更电影化，但每段都必须回到真实产品状态或真实产物。

### 平台能力验证视频

目标：证明视频生成、timeline、音频、字幕、artifact 链路是否可用。

结构：

1. 工具 gate。
2. 最小 clip 生成。
3. timeline 合成。
4. 下载 artifact。
5. 媒体探针和视觉检查。
6. 可用 / partial / blocked 结论。

重点：不要把“平台返回 succeeded”当作最终成功；必须检查真实文件。

### Blocked 导演稿

目标：当平台能力不足时，仍交付下一步可执行的高质量方案。

结构：

1. 阻塞层。
2. 证据。
3. 已完成素材或 clip。
4. 可执行 shot list。
5. 生成 prompt。
6. timeline / overlay / music plan。
7. 最小解除阻塞动作。

重点：不能编造成片能力；blocked 也要让下一位执行者能继续。

## Core Principles

1. **产品认知先于 case**
   - 开头 5-10 秒先让观众知道产品是什么、在哪工作、解决什么问题。
   - case 只能作为能力证明，不要让观众只记住 case 而忘了产品。

2. **每个镜头都要有产品价值**
   - 一个镜头对应一个产品问题：入口、上下文、编排、工具、状态、产物、审批、部署。
   - 纯气氛镜头最多作为 1-2 秒转场，不作为主要镜头。

3. **真实 UI 是主角**
   - 目录 tab、导航、面板标题、状态标签、artifact 标题必须来自真实产品或确定性 overlay。
   - 可以用景深、玻璃、光轨和空间层次增强，但 UI 结构要清楚可辨。

4. **视频模型做运动，后期做文字**
   - 模型适合生成慢推、景深、光线、流动、转场和产品空间感。
   - 产品名、步骤、关键 UI 文案、合规状态和 CTA 必须通过 deterministic overlay/timeline 控制。

5. **参考片拆机制，不拆表象**
   - Apple 类参考：产品实体、材质微距、暗场光、低饱和、节奏克制、结尾强收束。
   - Google Workspace 类参考：真实任务路径、用户输入、产品 UI 输出、跨工具串联、价值文案短而明确。
   - 禁止把这些变成抽象星图、未来城市、机器人拟人或随机英文 UI。

## Shot Architecture

产品宣传 + 轻教程混合片可使用 45-60 秒结构：

| Time | Role | Product Question | Visual Pattern |
| --- | --- | --- | --- |
| 0-6s | Product identity | 这是什么产品？ | 产品名/Studio 总览，慢推或慢拉，少字标题 |
| 6-12s | Entry | 从哪里开始？ | Chat 入口，用户输入或任务 prompt |
| 12-20s | Context | 它如何理解资料？ | Files / Knowledge / source context / provenance |
| 20-30s | Orchestration | Agent 如何执行？ | Agent Preset、runtime thread、tool calls、status trace |
| 30-40s | Work output | 产物如何出现？ | Artifacts、多渠道输出、版本或变体 |
| 40-50s | Control | 人如何 review 和批准？ | Review / refine / approve gate、状态变化 |
| 50-60s | System view | 为什么可信且可生产？ | Studio overview、Skills / Tools / Deploy、Logo end card |

如果当前不是产品宣传 + 轻教程混合片，不要硬套这张表；先回到 `Creative Variables` 和 `Scenario Outlines` 选择对应纲要。

如果需要补充 case，插入 5-8 秒“网页/文件/工具上下文读取”即可；它必须服务主线，不能另开故事。

## Camera And Motion Patterns

- 镜头语言必须服务类型：教程需要可读和停顿，品牌片需要情绪和细节，纪录片需要真实感，发布片需要节奏 build-up，社媒需要前段抓力。
- **Slow push-in**：用于建立产品实体或工作台可信感。
- **Horizontal glide**：用于从 Chat 移动到 Studio/Agent/Tools，表达工作流串联。
- **Layer reveal**：前景为真实 UI，中景为状态/工具，远景为 Studio 总览。
- **Action-driven transition**：用户发送、工具调用、artifact ready、approve 这些产品动作触发转场。
- **Hold for readability**：含 UI 文字的镜头至少停留到观众能识别主标题和状态，不用连续闪切。
- **End pull-back**：最后从细节拉回系统总览和产品名，形成发布片收束。

## Overlay Rules

关键 overlay 应短、确定、可读：

- 2-5 个词优先，例如 `Start with Chat`、`Add source context`、`Call tools`、`Review and approve`。
- 产品真实词优先，例如 `Chat`、`Agent Presets`、`Prompt`、`Knowledge`、`Tools`、`Skills`、`Deploy`、`Artifacts`。
- 避免解释型长句覆盖 UI；复杂解释放旁白或最终文案，不放画面中心。
- 字幕和状态标签必须不遮挡真实 UI 主路径。

## Generation Prompt Pattern

每个 shot 的 prompt 应包含：

```text
目标镜头：
真实素材：
产品价值：
镜头运动：
可见 UI 锚点：
确定性 overlay 计划：
风格约束：
negative prompt：
```

写 prompt 时要明确：

- “UI stays sharp and recognizable.”
- “Do not invent navigation labels.”
- “Readable product text is provided by overlay, not generated in-scene.”
- “No abstract sci-fi, no humanoid robots, no random English UI.”

## Tool And Timeline Gate

调用视频平台或 MCP workflow 前，先确认：

- 单次生成支持的时长，例如 5 秒 / 10 秒 / 长片。
- 是否支持 start image、多图、video-to-video、音频、字幕、timeline。
- 成本估算、质量报告、source manifest、approval state 是否存在。
- `render_video` 是生成模型还是确定性合成器。
- 是否能真正串联多个 clips；不能只看工具返回的 `duration` 字段。

如果工具返回成品，必须用媒体探针校验：

```bash
ffprobe -hide_banner -v error \
  -show_entries format=duration,size,bit_rate \
  -show_entries stream=index,codec_type,codec_name,width,height,avg_frame_rate,duration \
  -of json /path/to/final.mp4
```

校验结论分层：

- **Artifact valid**：真实时长、分辨率、音视频轨满足目标。
- **Partial**：clip 生成成功，但长片合成、音频或 overlay 不完整。
- **Blocked**：平台能力、工具 schema、成本/审批或素材权限缺失。

## Maxwell / Aion Specific Lessons

从 Maxwell Studio 视频生成任务中沉淀的通用教训：

- `generate_video` 只能生成短 clip 时，不要让它承担长片结构；先分 shot，再 timeline。
- 如果 `render_video` 声称 60 秒但实际 artifact 只有 10 秒，必须以本地媒体校验为准。
- 不要把 video clip URL 误传为 `audio_url`；没有真实音乐时先省略，或先用音频生成工具。
- 如果 timeline renderer 不能串联多 clip，返回 structured blocked，而不是继续声称成功。
- 用户要求“通过平台做”时，优先用平台 Chat/MCP 路径生成；本地 ffmpeg 只能作为校验或用户明确允许的补救合成。

## Review Checklist

交付前检查：

- 是否先选择了视频类型、风格和观看场景？
- 叙事、镜头语言、光线、色彩、声音和代入感是否服务当前视频，而不是套一个通用高级模板？
- 观众能在前 10 秒知道产品名、产品类别和工作场景吗？
- 每个主要镜头是否绑定了真实产品 UI 或真实信息架构？
- case 是否服务产品认知，而不是抢走主线？
- 是否存在随机 UI 文案、乱码、虚构 tab、机器人拟人或抽象科幻转场？
- 关键文字是否由 deterministic overlay 控制？
- 是否检查了真实时长、分辨率、帧率、音视频轨和文件大小？
- 如果 blocked，是否给出了阻塞层、证据和下一步最小动作？

## Anti-Patterns

- 把产品宣传片做成纯教程录屏，缺少品牌和系统认知。
- 把教程 case 做成唯一目的，观众看完不知道产品平台是什么。
- 把所有视频都做成同一种暗色科技 UI 风格，不管它原本需要纪录感、生活感、品牌情绪或社媒节奏。
- 大量抽象光效、星图、未来城市、AI 粒子，真实 UI 只出现一瞬间。
- 让视频模型生成长段可读文字，导致乱码或不可控。
- 工具返回成功后不下载、不探针、不检查真实媒体属性。
- 参考片只学“酷炫”和色调，没有学它的产品叙事和信息层级。
